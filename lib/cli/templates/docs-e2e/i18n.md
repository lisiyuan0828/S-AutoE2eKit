# 多语言 / i18n

> 这个文件告诉 skill：**项目支持哪些语种、key → 文案 怎么对、测试时怎么切换语种**。
>
> 如果你的项目目前只有中文 / 只有英文，可以保留这份模板，但**不必填**——
> skill 检测到这里只剩占位会跳过 i18n 相关测试。

## 1. 支持语种

| code | 名称 | 默认 |
| --- | --- | --- |
| `zh-CN` | 简体中文 | ✓ |
| `en-US` | 英文 | |

## 2. 切换方式

- 通过 URL query：`?lang=en-US`
- 或通过本地存储：`localStorage.setItem('lang', 'en-US')`
- 或通过头部下拉：`[data-testid="lang-switch"]` → 选项 `[data-testid="lang-option-en-US"]`

> skill 默认按"URL query"切，保证每个用例独立。

## 3. 关键文案对照

> 仅登记**测试断言**会用到的文案。普通 UI 文本不需要在这里登记。

| key | zh-CN | en-US |
| --- | --- | --- |
| `login.title` | 登录 | Sign in |
| `login.submit` | 登录 | Sign in |
| `login.errorWrongPassword` | 密码错误 | Wrong password |
| `order.success.title` | 下单成功 | Order placed |

## 4. 写法约定

- 断言文案时**优先用 i18n key**，让 skill 帮你翻译成当前语种：
  ```
  ✅ 期望页面显示 i18n("order.success.title")
  ❌ 期望页面显示 "下单成功"           // 英文跑会断
  ```
- 如果某个文案没在表里，请按 key 命名规则补一行，再让 skill 跑测试。
